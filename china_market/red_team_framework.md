# 红队攻击框架 (Red Team Attack Framework)

红队攻击是整套投研体系的"最后防线"。如果红队攻击只是泛泛地说"我可能高估了"，那和没有红队没有区别。

**本框架的铁律：每一条红队攻击，必须同时给出【攻击维度】+【具体证伪条件】+【触发时间节点】+【如果发生的应对策略】。缺少任何一项，Firewall 判定不通过。**

---

## 强制执行的七维攻击清单

每份报告的红队攻击部分，必须逐一回答以下七个维度。如果某个维度确实不适用（如公司无海外业务则地缘风险不适用），必须显式标注"本维度不适用，原因：XXX"，禁止静默跳过。

### 维度一：事实层攻击（数据可能是错的）

**问题**：报告中引用的核心数据，哪一条最有可能是过时的、被误读的、或被刻意美化的？

**输出格式**：
```
[攻击] 报告采信的 2026Q1 扣非净利润增速 63.44%，可能因为 [去年同期基数极低/一次性政府补贴] 而被高估。
[证伪条件] 若 2026 中报（预计 8 月底披露）扣非净利润同比增速回落至 20% 以下，则"业绩加速"的核心逻辑不成立。
[时间节点] 2026-08-31（中报披露截止日）
[应对策略] 立即将评级从 S 降至 B，执行减仓至半仓。
```

### 维度二：逻辑层攻击（推理链条可能是断的）

**问题**：报告中从"事实 A"推导出"结论 B"的逻辑链条中，最脆弱的环节在哪里？

**输出格式**：
```
[攻击] 报告推导"在手订单 64 亿 → 未来 3 年高增长确定"，但忽略了 [订单可能被延迟/取消/降价] 的可能性。
[证伪条件] 若 2026 年内出现单笔超过 5 亿的订单取消或重大客户流失公告，则推理链断裂。
[时间节点] 持续监控（每季度核查公告）
[应对策略] 一旦出现，立即触发 Firewall 重审，重新打分。
```

### 维度三：竞争层攻击（护城河可能在收窄）

**问题**：公司当前的竞争优势（技术壁垒/客户锁定/规模效应），最可能被谁、以什么方式颠覆？

**输出格式**：
```
[攻击] 竞争对手 [派克新材/中航重机] 如果在 [某项认证/某个客户] 上取得突破，将直接威胁公司的 [某项核心收入来源]。
[证伪条件] 若竞争对手获得 [GE/罗罗] 的同类型号认证或中标公告。
[时间节点] 持续监控
[应对策略] 重新评估护城河宽度，下调景气度维度得分。
```

### 维度四：资金层攻击（筹码可能崩塌）

**问题**：当前看好的资金面结构（机构加仓/北向新进），在什么情况下会突然反转？

**输出格式**：
```
[攻击] 公募基金 [某基金] 的加仓可能只是被动的指数调仓而非主动看好，一旦季末调仓结束，可能出现 [获利了结/仓位再平衡] 式抛售。
[证伪条件] 若下一个季报（2026Q2）披露该基金已将持仓削减 50% 以上。
[时间节点] 2026-07-15（基金季报披露日）
[应对策略] 资金维度得分从 9 分降至 4 分，总评级可能降档。
```

### 维度五：宏观层攻击（系统性风险）

**问题**：有哪些宏观级别的黑天鹅事件，会让本报告的所有假设同时失效？

**输出格式**：
```
[攻击] 若美联储意外加息或中美关系急剧恶化导致 [外资大规模撤离/港股暴跌/A股流动性危机]，则本报告所有基于"正常市场环境"的假设均将失效。
[证伪条件] 沪深 300 单月跌幅超过 15% 或北向资金单月净流出超 500 亿。
[时间节点] 不可预测（黑天鹅性质）
[应对策略] 启动全面防御模式，所有持仓降至最低仓位，等待企稳信号。
```

### 维度六：估值层攻击（可能买贵了）

**问题**：我给出的"估值合理/偏低"的判断，在什么条件下会被证明是错误的？

**输出格式**：
```
[攻击] 当前 PE 50 倍看似合理（因为增速高），但如果 [行业整体杀估值/板块风格切换到低估值红利]，则即使基本面不变，股价也可能因为估值中枢下移而跌 30%+。
[证伪条件] 若同类可比公司（如派克新材、三角防务）的 PE 中枢降至 30 倍以下，则航宇的 50 倍 PE 将显得极度昂贵。
[时间节点] 持续监控行业估值中枢
[应对策略] 估值维度得分归零，总评级自动降至 B 级。
```

### 维度七：执行层攻击（散户可能做不到）

**问题**：本报告给出的操作建议，散户在 A 股/港股的交易制度下，真的能执行吗？
**【强制定义】执行层攻击专指投资策略落地的“制度性/流程性障碍”（如：T+1导致跌停无法卖出、融券标的无券可借/成本过高、大宗交易价差过大、港股流动性枯竭导致挂单不成交）。严禁将宏观业务或财务风险写在执行层！**

**输出格式**：
```
[攻击] 报告建议"跌破 20 日线止损"，但在 A 股 T+1 制度下，如果当天跌破且次日低开跳空，散户的实际止损位会远低于理论止损价。科创板 20% 涨跌幅限制下，单日亏损可能直接达到 -15% 以上。
[证伪条件] 不适用（这是制度性约束，必然存在）
[时间节点] 始终有效
[应对策略] 在操作指令中补充"最大单日可承受亏损 = -X%，超过则无条件次日开盘清仓"的硬性止损规则。
```

## Firewall 红队攻击质量检查

在 Firewall 终检时，红队攻击部分必须通过以下检查：

| 检查项 | 通过标准 |
|--------|---------|
| 七维覆盖 | 7 个维度全部有输出（含"不适用"声明） |
| 具体性 | 每条攻击都包含具体的公司名/数字/时间，而非"可能出问题" |
| 证伪条件 | 每条攻击都有明确的、可观测的证伪条件 |
| 时间节点 | 每条攻击都有明确的验证时间窗口 |
| 应对策略 | 每条攻击都有对应的降级/减仓/清仓预案 |

**任何一项不通过，则报告退回重写红队攻击部分。**
