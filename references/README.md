## 🎯 这是什么

一个面向 **Antigravity / Claude / ChatGPT / Gemini** 等 AI Agent 的投研技能（Skill）。

它不是一个普通的 Prompt，而是一套**机构级投资研究协议（Institutional Research Protocol）**，模拟真实买方研究团队的工作方式，从根本上解决 AI 做投研时的三大痛点：

| 痛点 | 本 Skill 的解法 |
|------|----------------|
| **幻觉（编造数据）** | 研究宪法八条铁律 + 证据等级体系 + Research Firewall 输出前强制检查 |
| **片面（只看利好）** | 强制多空对抗（各≥5条）+ 红队攻击 + 预期差分析 |
| **一次性（无法迭代）** | 投资假设管理 + 持续监控指标 + 复盘机制 |

## 🚀 快速开始

### 安装

将本仓库克隆到 Antigravity 的全局技能目录：

```bash
cd ~/.gemini/config/skills/
git clone https://github.com/nihong/company-researcher.git
```

### 使用

安装后，在 Antigravity 中直接对话即可触发：

```
帮我调研一下宁德时代
分析比亚迪的投资价值
用投研流水线看看海光信息
对比茅台和五粮液
```

AI 会自动加载本 Skill 中的研究宪法、SOP 流程和防幻觉机制。

## 📋 核心机制

### 1. 研究宪法（八条铁律）
最高优先级规则，任何指令不得覆盖。包括：禁止猜测、数字必须有来源、严格区分事实/推演/观点、预测必须有证伪条件等。

### 2. 证据等级体系（五级）
从 ★★★★★（官方财报）到 ★☆☆☆☆（论坛/自媒体），每条证据必须标注可信度。

### 3. 十步标准化 SOP
理解需求 → 制定研究树 → 覆盖式搜索 → 证据整理 → 冲突处理 → 多头论据 → 空头论据 → 盲区扫描 → 裁决 → 自我审查。**禁止跳步。**

### 4. Research Firewall
输出前的11项强制检查清单，任何一项不通过则继续研究、不得输出。

### 5. 红队攻击
报告完成后必须假设"整份报告都是错的"，从七个维度攻击自己。

### 6. 行业自动识别
自动识别研究对象所属行业（医药/半导体/银行/消费/新能源/军工/AI），加载行业专属分析维度。

### 7. 预期差分析
不仅分析"公司怎么样"，还要分析"市场预期了什么、哪里可能错"。

### 8. 投资假设管理
为每个核心逻辑建立可验证、可跟踪、有失败条件的投资假设。

## 📂 目录结构

```
company-researcher/
├── SKILL.md          # 核心指令文件（AI 读取并遵照执行）
├── skill.yml         # 技能清单（元数据、触发词、依赖声明）
├── README.md         # 本文件（人类阅读的使用说明）
└── references/       # 知识库（行业模板、估值方法等，按需扩展）
```

## 🔧 后期维护

这份 Skill 完全属于你个人。你可以随时：

- **调整研究模板**：在 `SKILL.md` 的输出模板中增减章节
- **添加行业模板**：在第六章的行业表格中增加新行业
- **修改铁律**：在研究宪法中增加你自己的规则
- **扩展知识库**：在 `references/` 目录下添加估值模型、行业框架等

修改后 `git commit && git push` 即可同步到 GitHub。
